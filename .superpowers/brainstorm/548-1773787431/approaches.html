A — Refined Evolution (Recommended)

Modernize within the existing Astro + CSS-only architecture. Replace the warm palette with monochrome, add frosted glass surfaces with backdrop-filter, enhance hover states with CSS transforms/transitions. New landing page as a simple index.astro route. All pure CSS — no new dependencies.

Pros

  • Zero new dependencies
  • Fastest to implement
  • Keeps build tiny (~400K)
  • All existing tests still work

Cons

  • Micro-interactions limited to CSS (no 3D tilt, magnetic cursor)
  • Page transitions stay basic (Astro ViewTransitions)

B — Full Redesign with Motion Library

Same Astro stack but add a lightweight animation library (Motion One ~3KB or CSS-only advanced animations). Centered floating pill nav, 3D card tilts on hover, magnetic buttons, staggered grid reveals, smooth page morphs via View Transitions API. More immersive but still lean.

Pros

  • Full micro-interaction suite (3D tilt, magnetic, spring physics)
  • More polished, Apple/Notion-level feel
  • Still static output, no heavy JS bundle
  • View Transitions API for page morphs

Cons

  • Adds ~3-5KB JS for animation helpers
  • More CSS complexity
  • Need to update e2e tests for new interactions
